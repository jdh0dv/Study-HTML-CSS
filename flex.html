flex가 나오기 전, 박스모델의 가로정렬을 위해 inline-block이나 float요소를 통해 레이아웃을 지정했다.
inline-block의 경우 자동으로 그리드가 형성되지 못하고 정적이며, float의 경우 다른 박스모델에도 영향을 준다.
이에 반해 flex는 유연하고 다른 박스모델에도 영향을 안주기 때문에 반응형이나 모바일웹 환경에서 매우 유용하다.

[부모 영역에 쓰이는 속성값]

  1. display : container(자식들의 부모영역)를 정의

    • flex : block 특성의 flex container
    • flex-inline : lnline 특성의 flex container

  2. justify-content : 축의 방향

  3. align-items : 축의 수직방향

  4. align-content : *flex-wrap:wrap인 상태에서 가능

  5. flex-direction : 배치방향 전환 (default:row)

  6. flex-wrap : 줄바꿈을 허용할 것인지 결정

[자식 영역에 쓰이는 속성값]

  1. flex-grow : 여백 조정하는 값

  2. flex-basis : 자식의 점유공간을 조절(여백 부여)

  3. flex-shrink :

  4. flex : basis+shrink 축햑형 *숫자로 작성 (신축성 있음)

  5. align-self : 각각 위치를 조정 (개별로)

  6. order : 아이템들의 순서 *숫자로 작성
